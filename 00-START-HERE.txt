🔐 SEGURIDAD - IMPLEMENTADO
═══════════════════════════════════════════════════════════════════════════════════

BIOMETRÍA IRREVERSIBLE:
  Imagen (200KB) → SHA-256 + Salt → Hash (64 caracteres)
  ❌ Imagen NUNCA se persiste
  ❌ Hash NO se puede revertir
  ✅ Idénticas imágenes = mismo hash (verificación)

VALIDACIÓN RUT CHILENO:
  12.345.678-9 → Módulo 11 → Verify checksum
  ✅ Algoritmo correcto (pesos 2-7)
  ✅ Soporta dígito K
  ✅ Normalización automática

ENCRIPTACIÓN AES-256:
  Datos sensibles → AES-256-GCM + IV único → Base64
  ✅ 256 bits de seguridad
  ✅ Integridad verificada
  ✅ IV único por registro

AUDITORÍA (LEY 19.628):
  Cada evento registra:
  ✅ Quién: userId
  ✅ Cuándo: timestamp UTC
  ✅ Qué: eventType (ENROLLMENT_START, etc)
  ✅ Dónde: ipAddress del cliente
  ✅ Cómo: userAgent del navegador

═══════════════════════════════════════════════════════════════════════════════════
📊 ENDPOINTS API
═══════════════════════════════════════════════════════════════════════════════════

1. POST /api/enrollment/initiate
   Request:  { rut, email, fullName }
   Response: { userId, enrollmentStatus: "PENDING" }

2. POST /api/enrollment/capture-biometric
   Request:  { userId, facialImage, biometricType }
   Response: { userId, enrollmentStatus: "BIOMETRIC_CAPTURED" }

3. POST /api/enrollment/sign-consent
   Request:  { userId, acceptsPrivacyTerms, acceptsBiometricProcessing, ... }
   Response: { userId, enrollmentStatus: "ACTIVE", isConsentSigned: true }

4. GET /api/enrollment/status/:userId
   Response: { userId, rut, email, enrollmentStatus, ... }
